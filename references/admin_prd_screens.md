## 🎨 화면별 상세 기획

### 1. 로그인 화면

#### UI 구성 (Ant Design 컴포넌트)
- **Card**: 중앙 로그인 박스
- **Button** (primary, large): Google 로그인 버튼
- **Space**: 수직 간격 정렬
- **Typography**: 제목과 설명 텍스트
- **Icon**: GoogleOutlined

#### 와이어프레임
```
┌─────────────────────────────────┐
│                                 │
│         [로고]                  │
│    Portfolio Admin              │
│                                 │
│    포트폴리오를 관리하세요        │
│                                 │
│  [🔐 Google로 로그인]           │
│                                 │
└─────────────────────────────────┘
```

#### 기능
- Google OAuth 로그인
- 로그인 성공 시 대시보드로 이동
- 로그인 실패 시 `message.error()` 표시

### 3. 작업 목록

#### Ant Design 컴포넌트
- **Table**: 작업 목록 테이블 (rowSelection으로 체크박스)
- **Button**: 새 작업, 편집, 삭제 버튼
- **Input**: 검색 입력창
- **Select**: 필터 드롭다운
- **Switch**: 공개/비공개 토글
- **Tag**: 카테고리 태그
- **Image**: 썸네일 이미지
- **Space**: 버튼 그룹 정렬
- **Card**: 일괄 작업 영역

#### 와이어프레임
```
┌──────────────────────────────────────────────────────────┐
│  🎨 작업 관리                                            │
├──────────────────────────────────────────────────────────┤
│  [+ 새 작업]  [필터: 전체 ▼]  [검색: ____________]  [미리보기]│
├──────────────────────────────────────────────────────────┤
│                                                          │
│  Table 컴포넌트:                                         │
│  ┌────────────────────────────────────────────────────┐ │
│  │ [✓] [썸네일]  작품명1              [디자인]         │ │
│  │              간단한 설명...         ● 공개          │ │
│  │              2025.10.15            [편집] [삭제]    │ │
│  ├────────────────────────────────────────────────────┤ │
│  │ [✓] [썸네일]  작품명2              [조각][추상]     │ │
│  │              간단한 설명...         ○ 비공개        │ │
│  │              2025.10.10            [편집] [삭제]    │ │
│  └────────────────────────────────────────────────────┘ │
│                                                          │
│  Card (일괄 작업):                                       │
│  선택된 항목 (2개): [일괄 공개] [일괄 비공개] [카테고리 추가]│
│                                                          │
│  Pagination: [1] [2] [3] ... [10]  →                    │
└──────────────────────────────────────────────────────────┘
```

#### 기능

**필터링**:
- 전체 / 공개 / 비공개
- 카테고리별 (다중 선택 가능)

**검색**:
- 작업 제목으로 검색
- 실시간 검색 (500ms 디바운스)

**정렬**:
- 최신순 (기본)
- 오래된순
- 제목순 (가나다)

**일괄 작업**:
- 체크박스로 여러 작업 선택
- 선택한 작업들을 한 번에:
  - 공개/비공개 변경
  - 카테고리 추가/제거
  - 삭제

**공개/비공개 토글**:
- 목록에서 바로 토글 가능 (Switch 컴포넌트)
- 클릭 시 즉시 반영 (Optimistic Update)
- 성공 시 토스트 메시지

**미리보기**:
- 우측 상단 [미리보기] 버튼
- 새 탭에서 실제 사용자 페이지로 이동

**성능 최적화** ⚡:
- **가상 스크롤**: 100개 이상 작업도 빠른 렌더링
- **이미지 Lazy Loading**: 화면에 보이는 썸네일만 로드
- **Pagination**: 20개씩 로드, 무한 스크롤 지원
- **캐싱**: React Query로 이미 본 목록은 즉시 표시
- **메모이제이션**: 불필요한 리렌더링 방지

#### 기능 상세

##### 4.1 기본 정보 입력

**제목 (필수)**:
- 최대 100자
- 실시간 글자 수 표시
- 빈 값 제출 시 에러

**간단한 설명 (선택)**:
- 최대 200자
- 카드 UI에 표시될 내용
- 없으면 상세 설명에서 자동으로 일부 추출

**상세 설명 (필수)**:
- 최대 5000자
- 일반 텍스트 입력 (마크다운 미지원)
- 줄바꿈 가능

**공개 상태**:
- 라디오 버튼: 공개 / 비공개
- 기본값: 비공개

##### 4.2 이미지 업로드 및 관리

**이미지 카드 UI**:
```
┌─────────────┐
│    [순서]   │
│   1         │
│  ┌───────┐  │
│  │       │  │
│  │ IMG   │  │
│  │       │  │
```

**순서 변경**:

- 최대 영상 개수: 10개 (작업당)

##### 4.3 작업 캡션 입력 (페이지당 1개)

**캡션 에디터**:
- 작업(페이지)당 하나의 캡션만 입력 가능
- WYSIWYG 스타일 (Tiptap)
- 최대 1000자
- 해당 작업의 전체 이미지들에 대한 설명을 하나의 캡션으로 작성

**에디터 툴바**:
```
[B] [I] [U] [🔗 작업 링크]
```
- **B**: Bold (진하게)
- **I**: Italic (기울임)
- **U**: Underline (밑줄)
- **🔗**: 작업 링크 삽입

**작업 링크 삽입 플로우**:
```
1. 캡션에서 텍스트 드래그 선택
   예: "아름다운 바다"

2. [🔗 작업 링크] 버튼 클릭

3. 모달 팝업
   ┌────────────────────────────┐
   │ 작업 검색                  │
   │ [검색: _____________]      │
   │                            │
   │ 검색 결과:                 │
   │  [썸네일] 아름다운 바다     │
   │  [썸네일] 아름다운 하늘     │
   │  [썸네일] 푸른 바다        │
   │                            │
   │      [취소]  [선택]        │
   └────────────────────────────┘

4. 작업 선택

5. 자동으로 HTML 링크 생성
   <a href="/works/work-123" data-work-id="work-123">
     아름다운 바다
   </a>

6. 캡션에 삽입
```

**작업 링크 표시**:
- 에디터 내에서 링크는 파란색 + 밑줄
- 호버 시 툴팁으로 작업명 표시
- 클릭 시 링크 수정 가능

**📱 사용자 페이지에서 작업 링크 동작 (참고)**:
관리자가 삽입한 작업 링크는 사용자 페이지에서 다음과 같이 동작합니다:
- **표시**: 진한 명도로 강조 표시
- **hover**: 500ms 후 Floating Window로 프리뷰 (제목 + 썸네일)
- **click**: **모달 창으로 작업 상세 페이지 표시**
  - 모달 구조: 상단(작업명) + 좌측(이미지들) + 우측(캡션 고정)
  - 모달 내 캡션에도 작업 링크 삽입 가능
  - 모달 내 작업 링크 클릭 시: 현재 모달 닫힘 → 새 모달로 해당 작업 표시
  - 재귀적으로 다른 작업으로 이동 가능

- 괄호 안에 전체 문장 표시

- 사용자 페이지에서 키워드 클릭 시 해당 작업들 표시
- **다른 문장 간 키워드 중복 선택 가능**: 하나의 작업이 여러 문장의 키워드에 동시에 속할 수 있음

##### 4.5 저장 및 상태 관리

**임시 저장**:
- [임시저장] 버튼
- 현재 입력한 내용을 서버에 저장 (비공개 상태)
- 성공 시 토스트: "임시 저장되었습니다"
- 언제든 [작업 수정]으로 돌아와서 이어서 작업 가능

**미리보기**:
- [미리보기] 버튼
- 새 탭에서 실제 사용자 페이지 렌더링
- 저장하지 않은 변경사항도 미리보기 가능 (임시 URL 생성)

**저장**:
- [저장] 버튼
- 필수 항목 검증:
  - 제목 (필수)
  - 상세 설명 (필수)
  - 이미지 최소 1장
  - 대표 썸네일 선택
- 검증 실패 시:
  - 해당 필드로 스크롤
  - 에러 메시지 표시
- 검증 성공 시:
  - 서버에 저장
  - 작업 목록으로 이동
  - 성공 토스트 표시

**취소**:
- [취소] 버튼
- 변경사항이 있으면 확인 모달:
  ```
  "저장하지 않은 변경사항이 있습니다.
   정말 나가시겠습니까?"
   [임시저장] [나가기] [계속 작업]
  ```

**자동 저장** (선택 사항):
- 5분마다 자동으로 임시 저장
- 화면 상단에 "자동 저장됨 (5분 전)" 표시

- **점 위치**: 각 이미지의 좌측 상단 코너에 정렬
- **스크롤 동작**: Snap 효과 + Interpolation 애니메이션
- 관리자가 설정한 이미지 순서대로 타임라인에 표시됨
