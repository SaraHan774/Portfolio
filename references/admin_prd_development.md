## 🎯 핵심 사용자 시나리오

### 시나리오 1: 전시 끝나고 작품 일괄 업로드 (데스크탑)

**목표**: 전시에서 찍은 작품 사진 50장을 빠르게 업로드

**단계**:
1. 관리자 페이지 로그인
2. [+ 새 작업] 클릭
3. 제목 입력: "2024 졸업 전시 - 작품 1"
4. 이미지 업로드 영역에 10장 드래그 앤 드롭
5. 업로드 중... (진행률 표시)
6. 대표 썸네일 선택 (첫 번째 이미지)
7. [임시저장] → 다음 작품으로
8. 10개 작품 반복 업로드
9. 나중에 카테고리와 캡션 추가

**소요 시간**: 작품 1개당 2-3분 (10개 작품 약 30분)

---

### 시나리오 2: 공모전 지원 전 작품 순서 조정 (데스크탑)

**목표**: 대표 작품들을 앞으로 배치

**단계**:
1. [카테고리 관리] → [텍스트형 카테고리]
2. "대표작" 카테고리 선택
3. [순서 변경] 클릭
4. 드래그로 작품 순서 조정
   - 가장 잘 나온 작품을 맨 앞으로
   - 시리즈물은 연속으로 배치
5. [저장]
6. [미리보기]로 확인

**소요 시간**: 5분

---

### 시나리오 3: 작품에 캡션 추가하며 다른 작품 링크 걸기 (데스크탑)

**목표**: 작품 설명에 관련 작품 링크 추가

**단계**:
1. [작업 관리] → 수정할 작품 선택
2. 작업 캡션 에디터에 입력:
   "이 작품은 <드래그>아름다운 바다</드래그> 시리즈에서 영감을 받았습니다"
3. "아름다운 바다" 텍스트 선택 상태에서
4. [🔗 작업 링크] 버튼 클릭
5. 검색창에 "바다" 입력
6. 검색 결과에서 "아름다운 바다" 작품 선택
7. 자동으로 링크 생성됨
8. [저장]
9. [미리보기]에서 링크 hover 동작 확인

**소요 시간**: 2분

---

### 시나리오 4: 🆕 외출 중 급하게 작품 정보 수정 (모바일)

**목표**: 카페에서 스마트폰으로 작품 공개 상태 변경 및 설명 수정

**단계**:
1. 모바일 브라우저에서 관리자 페이지 접속
2. Google 로그인 (자동 로그인 유지)
3. 햄버거 메뉴 → [작업 관리]
4. 작품 목록에서 해당 작품 카드 탭
5. 수정 화면 진입
6. 공개 상태 토글 (비공개 → 공개)
7. 설명 텍스트 수정 (모바일 키보드 사용)
8. 하단 [저장] 버튼 탭
9. 성공 메시지 확인

**소요 시간**: 1-2분

---

### 시나리오 5: 🆕 전시 현장에서 작품 사진 바로 업로드 (모바일)

**목표**: 전시 현장에서 스마트폰으로 촬영한 사진을 즉시 업로드

**단계**:
1. 스마트폰으로 작품 사진 5장 촬영
2. 관리자 페이지 접속 (PWA 앱으로 저장해두면 더 빠름)
3. 하단 FAB(+) 버튼 탭
4. 제목 입력: "전시 현장 - 작품명"
5. [이미지 추가] → [카메라로 촬영] 또는 [갤러리에서 선택]
6. 방금 찍은 5장 선택
7. 자동 업로드 시작 (진행률 표시)
8. 첫 번째 이미지를 대표 썸네일로 선택
9. [임시저장] (나중에 상세 설명 추가 예정)
10. 계속 촬영하며 다음 작품 업로드

**소요 시간**: 작품 1개당 3-4분

---

### 시나리오 6: 🆕 지하철에서 작업 순서 변경 (모바일)

**목표**: 출퇴근 중 스마트폰으로 카테고리 내 작업 순서 조정

**단계**:
1. 모바일에서 관리자 페이지 접속
2. 메뉴 → [카테고리 관리]
3. "대표작" 카테고리 선택
4. [순서 변경] 버튼
5. 작품 카드 길게 누르기 → 드래그 활성화
6. 원하는 위치로 이동
7. 또는 [위로]/[아래로] 버튼으로 순서 조정
8. [저장]
9. 자동으로 이전 화면 복귀

**소요 시간**: 3분

## 🎨 UI/UX 가이드라인

### 디자인 원칙

1. **단순함**: 불필요한 요소 제거
2. **명확함**: 버튼과 액션이 명확하게 보임
3. **피드백**: 모든 액션에 즉각적인 피드백
4. **일관성**: Ant Design 컴포넌트 활용으로 일관성 유지

### 컬러 시스템 (Ant Design 기본 테마)

```typescript
// Ant Design 기본 색상 토큰 사용
{
  colorPrimary: '#1890ff',      // 주요 액션
  colorSuccess: '#52c41a',      // 성공 상태
  colorWarning: '#faad14',      // 경고
  colorError: '#ff4d4f',        // 에러
  colorInfo: '#1890ff',         // 정보
  colorTextBase: '#000000',     // 기본 텍스트
  colorBgContainer: '#ffffff',  // 배경
}

// 커스터마이징 (선택)
// ConfigProvider로 전역 테마 변경 가능
```

### 타이포그래피 (Ant Design Typography 컴포넌트)

```tsx
<Typography.Title level={1}>헤딩 1</Typography.Title>  // 38px
<Typography.Title level={2}>헤딩 2</Typography.Title>  // 30px
<Typography.Title level={3}>헤딩 3</Typography.Title>  // 24px
<Typography.Title level={4}>헤딩 4</Typography.Title>  // 20px
<Typography.Text>본문</Typography.Text>                 // 14px
<Typography.Text type="secondary">캡션</Typography.Text> // 14px, 회색
```

### 간격 시스템 (Ant Design Space 컴포넌트)

```tsx
<Space size="small">    // 8px
<Space size="middle">   // 16px (기본)
<Space size="large">    // 24px

// 또는 직접 지정
<Space size={32}>       // 32px
```

### 레이아웃 컴포넌트

```tsx
<Layout>
  <Layout.Header />    // 헤더
  <Layout.Sider />     // 사이드바
  <Layout.Content />   // 메인 컨텐츠
</Layout>

<Row gutter={16}>      // 그리드 시스템
  <Col span={8}>
  <Col span={16}>
</Row>
```

### 반응형 브레이크포인트 (Ant Design Grid)

```tsx
// Ant Design 그리드 브레이크포인트
{
  xs: 480,   // < 576px (모바일)
  sm: 576,   // >= 576px (모바일)
  md: 768,   // >= 768px (태블릿)
  lg: 992,   // >= 992px (데스크탑)
  xl: 1200,  // >= 1200px (큰 데스크탑)
  xxl: 1600  // >= 1600px (초대형 화면)
}

// 사용 예시
<Col xs={24} sm={12} md={8} lg={6}>
```

## 📱 모바일 반응형 전략

### 기본 원칙
- **Mobile First**: 모바일 화면을 기준으로 설계 후 데스크탑으로 확장
- **터치 최적화**: 모든 버튼과 인터랙션 요소는 최소 44px × 44px
- **완전한 기능**: PC에서 할 수 있는 모든 기능을 모바일에서도 동일하게 제공
- **자연스러운 전환**: 디바이스 크기에 따라 레이아웃만 변경, 기능은 동일

### 화면별 모바일 대응

**1. 레이아웃 (전역)**
- **데스크탑**: Header + Sider(고정) + Content
- **모바일**:
  - Header (햄버거 메뉴 버튼)
  - Content (전체 화면)
  - Drawer (슬라이드 메뉴)
  - 하단 고정 FAB (Floating Action Button) - 빠른 작업 추가

**2. 대시보드**
- **데스크탑**: 통계 카드 3열 배치
- **모바일**: 통계 카드 1열 배치 (세로 스크롤)

**3. 작업 목록**
- **데스크탑**: Table 컴포넌트 (가로 스크롤)
- **모바일**:
  - List 컴포넌트로 전환 (카드 형식)
  - 스와이프 액션 (왼쪽 스와이프 → 편집/삭제 버튼)
  - 필터/검색은 상단 고정
  - 일괄 작업은 하단 Sheet 형식

**4. 작업 폼 (핵심!)**
- **공통**: 모든 섹션은 Card 단위로 접을 수 있음 (Collapse)
- **데스크탑**:
  - 좌우 고정 액션바 (임시저장, 미리보기, 저장)
  - 이미지 그리드 (4-5열)
- **모바일**:
  - 상단/하단 고정 액션바
  - 이미지 그리드 (2열)
  - 드래그 앤 드롭 → 길게 누르기 + 순서 변경 버튼
  - 이미지 회전/삭제는 이미지 탭 시 모달

**5. 이미지 업로드**
- **데스크탑**: 드래그 앤 드롭 + 클릭 업로드
- **모바일**:
  - 카메라 직접 촬영 (Input accept="image/*" capture="camera")
  - 갤러리에서 선택
  - 여러 장 선택 가능

**6. 캡션 에디터**
- **데스크탑**: 툴바 상단 고정
- **모바일**:
  - 키보드 위 툴바 표시
  - 작업 링크 삽입은 전체 화면 모달

**7. 카테고리 관리**
- **데스크탑**: 카드 목록
- **모바일**:
  - Collapse로 접기/펼치기
  - 순서 변경은 버튼으로 (위로/아래로)
  - 드래그는 길게 누르기로 활성화

### 모바일 전용 기능

1. **PWA (Progressive Web App) 지원**
   - 홈 화면에 추가 가능
   - 오프라인 기본 동작 (임시 저장)
   - 푸시 알림 (선택 사항)

2. **제스처 지원**
   - 스와이프로 작업 편집/삭제
   - Pull to refresh (작업 목록)
   - 두 손가락 확대/축소 (이미지 미리보기)

3. **모바일 최적화**
   - 이미지 자동 압축 (업로드 전)
   - 로딩 인디케이터 (느린 네트워크 대응)
   - Bottom Sheet (모바일 친화적 모달)

### 테스트 디바이스

- iPhone 14 Pro (390px)
- Samsung Galaxy S23 (360px)
- iPad (768px)

### 성공 기준
- ✅ 모든 기능이 모바일에서 동작
- ✅ 터치 타겟 최소 44px × 44px
- ✅ 세로 모드에서 한 손으로 조작 가능
- ✅ 모바일 사파리/크롬에서 정상 동작
- ✅ 사용자 테스트: "모바일로 작품 1개 업로드 가능"

## 🔮 향후 계획 (Phase 4+)

### 우선순위 높음

#### 1. **분석 기능** 📊
**목적**: 어떤 작품이 인기 있는지 파악하여 포트폴리오 개선

**기능**:
- **작품별 조회수**
  - 일별/주별/월별 조회수 그래프
  - 가장 많이 본 작품 TOP 10
  - 평균 체류 시간

- **카테고리별 인기도**
  - 카테고리별 클릭 수
  - 어떤 카테고리로 들어오는지 분석

- **방문자 통계**
  - 일별 방문자 수
  - 신규 vs 재방문자
  - 접속 시간대 분석

- **유입 경로**
  - 어디서 링크 타고 왔는지 (Referrer)
  - 직접 접속 vs 링크 공유

- **디바이스 분석**
  - 모바일 vs 데스크탑 비율
  - 브라우저 종류

**구현 방법**:
- Google Analytics 4 연동 (가장 간단)
- 또는 자체 분석 시스템 (Privacy 중시 시)
- 관리자 페이지에 대시보드 추가

---

#### 2. **SEO 최적화** 🔍
**목적**: 구글 검색에 작품이 노출되어 더 많은 사람이 볼 수 있도록

**기능**:
- **메타 태그 자동 생성**
  - 작품 제목 → `<title>` 태그
  - 간단한 설명 → `<meta description>`
  - 대표 이미지 → Open Graph 이미지

- **Open Graph / Twitter Card**
  - SNS 공유 시 예쁜 카드 표시
  - 작품 썸네일 + 제목 + 설명

- **구조화된 데이터 (Schema.org)**
  - CreativeWork 마크업
  - 작가 정보
  - 작품 정보

- **Sitemap 자동 생성**
  - XML Sitemap
  - 새 작품 추가 시 자동 업데이트
  - Google Search Console 제출

- **이미지 최적화**
