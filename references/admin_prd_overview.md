## 🎯 프로젝트 목표

### 핵심 가치
"기술을 모르는 미대생도 혼자서 포트폴리오를 관리할 수 있다"

### 주요 목표
1. **쉬운 작품 업로드**: 10장의 사진을 5분 안에 올릴 수 있어야 함
2. **직관적인 UI**: 설명서 없이도 사용 가능
3. **빠른 수정**: 급하게 작품 정보를 바꿔야 할 때 1분 안에 가능
4. **실수 방지**: 되돌리기, 임시 저장, 미리보기 기능
5. **완전한 반응형**: 모바일에서도 PC와 동일하게 모든 기능 사용 가능

## 👤 사용자 페르소나

### 주 사용자: 민지 (25세, 조소과 4학년)

**배경**:
- 졸업 전시 준비 중
- 작품 50개 정도 보유
- 공모전 지원용 포트폴리오 필요
- 코딩은 전혀 모름
- 노션, 인스타그램은 익숙함

**니즈**:
- "전시 끝나면 바로 사진 올려야 하는데..."
- "작품 순서 바꾸기 쉬웠으면..."
- "나중에 혼자서도 수정할 수 있어야 해"
- "실수로 삭제하면 큰일나는데..."

**좌절 포인트**:
- 복잡한 관리자 페이지 (워드프레스 같은 거)
- 이미지 업로드가 느림
- 카테고리 관리가 복잡함
- **모바일에서 제대로 안 됨** (급하게 외출 중에 수정할 때)

## 📈 성공 지표 (KPI)

### 사용성

- 작업 업로드 완료율: > 90%
- 평균 작업 생성 시간: < 5분
- 에러 발생률: < 1%

### 성능

- 페이지 로딩 시간: < 2초
- 이미지 업로드 성공률: > 95%
- 서버 응답 시간: < 500ms

### 사용자 만족도

- NPS (Net Promoter Score): > 50
- 사용자 리텐션 (7일): > 70%
- 평균 세션 길이: > 10분

## 📝 변경 이력

### v1.3 (2025-12-18)
- 영상 업로드 기능 추가 (YouTube Embed 활용)
- WorkVideo 인터페이스 및 MediaItem 타입 정의 추가
- 영상 추가 어드민 기능 (4.2.1 영상 관리 섹션)
- 이미지와 영상의 순서를 자유롭게 조합 가능하도록 미디어 관리 통합
- 와이어프레임 업데이트: 미디어 관리 UI에 영상 카드 추가
- 문장형 카테고리: 다른 문장 간 키워드 중복 선택 가능 스펙 추가

### v1.2 (2025-12-08)
- 카테고리 구조 변경: 텍스트형 카테고리 → 전시명 카테고리로 변경
- 전시명 카테고리 데이터 구조 추가 (작업명, 전시유형, 공간, 년도)
- 문장형 카테고리는 키워드 개별 클릭, 전시명 카테고리는 통으로 클릭
- SentenceCategory, ExhibitionCategory 인터페이스 정의 추가
- 사용자 페이지 표시 형태 참고 섹션 추가
  - 작업 카드 표시 형태 (제목 + 썸네일 아이콘)
  - 카테고리별 정렬 방향 (문장형: 좌→우, 전시명: 우→좌)
- 캡션 내 작업 링크의 사용자 페이지 동작 설명 추가
  - 모달 창으로 작업 상세 페이지 표시
  - 모달 내 캡션에서도 재귀적으로 작업 이동 가능
- 이미지 스크롤바 타임라인 UI 참고 섹션 추가

### v1.1 (2025-12-08)
- 캡션 구조 변경: 이미지별 캡션 → 작업(페이지)당 1개의 캡션으로 수정
- 관련 와이어프레임, 시나리오, Phase 2 항목 업데이트

### v1.0 (2025-11-02)
- 초기 PRD 작성
- 조소과 학생 페르소나 기반 기획
- Phase 1-3 개발 계획 수립
