# 포트폴리오 웹사이트 관리자 페이지 PRD

## 📋 문서 정보

- **문서 버전**: 1.0
- **작성일**: 2025년 11월 2일
- **대상 사용자**: 포트폴리오 작가 (조소과 학생 페르소나)
- **목적**: 작품을 쉽고 빠르게 업로드하고 관리할 수 있는 관리자 시스템

---

## 🎯 프로젝트 목표

### 핵심 가치
"기술을 모르는 미대생도 혼자서 포트폴리오를 관리할 수 있다"

### 주요 목표
1. **쉬운 작품 업로드**: 10장의 사진을 5분 안에 올릴 수 있어야 함
2. **직관적인 UI**: 설명서 없이도 사용 가능
3. **빠른 수정**: 급하게 작품 정보를 바꿔야 할 때 1분 안에 가능
4. **실수 방지**: 되돌리기, 임시 저장, 미리보기 기능
5. **완전한 반응형**: 모바일에서도 PC와 동일하게 모든 기능 사용 가능

#### 기능

**필터링**:
- 전체 / 공개 / 비공개
- 카테고리별 (다중 선택 가능)

**검색**:
- 작업 제목으로 검색
- 실시간 검색 (500ms 디바운스)

**정렬**:
- 최신순 (기본)
- 오래된순
- 제목순 (가나다)

**일괄 작업**:
- 체크박스로 여러 작업 선택
- 선택한 작업들을 한 번에:
  - 공개/비공개 변경
  - 카테고리 추가/제거
  - 삭제

**공개/비공개 토글**:
- 목록에서 바로 토글 가능 (Switch 컴포넌트)
- 클릭 시 즉시 반영 (Optimistic Update)
- 성공 시 토스트 메시지

**미리보기**:
- 우측 상단 [미리보기] 버튼
- 새 탭에서 실제 사용자 페이지로 이동

**성능 최적화** ⚡:
- **가상 스크롤**: 100개 이상 작업도 빠른 렌더링
- **이미지 Lazy Loading**: 화면에 보이는 썸네일만 로드
- **Pagination**: 20개씩 로드, 무한 스크롤 지원
- **캐싱**: React Query로 이미 본 목록은 즉시 표시
- **메모이제이션**: 불필요한 리렌더링 방지

- 없으면 상세 설명에서 자동으로 일부 추출

**상세 설명 (필수)**:
- 최대 5000자

- 줄바꿈 가능

**공개 상태**:
- 라디오 버튼: 공개 / 비공개
- 기본값: 비공개

##### 4.2 이미지 업로드 및 관리

**업로드 영역**:
```typescript
interface ImageUploadProps {
  maxFiles: 50;
  maxFileSize: 10 * 1024 * 1024; // 10MB
  accept: ['image/jpeg', 'image/png', 'image/webp'];
  multiple: true;
  onDrop: (files: File[]) => void;
}
```

**드래그 앤 드롭**:
- 파일 탐색기에서 드래그 → 업로드 영역에 드롭
- 여러 파일 동시 업로드 가능
- 업로드 중 진행률 표시 (0% → 100%)

**업로드 프로세스**:
```
1. 파일 선택/드롭
   ↓
2. 클라이언트 측 검증
   - 파일 형식 체크
   - 파일 크기 체크
   - 최대 개수 체크
   ↓
3. 프리뷰 생성 (로컬)
   ↓
4. 서버 업로드
   - 진행률 표시
   - 썸네일 자동 생성 (1:1)
   ↓
5. 완료
   - 이미지 목록에 추가
   - 순서는 업로드 순서
```

```
│  (●) 대표    │  ← 라디오 버튼 (하나만 선택)
│  [↻ 회전]   │  ← 90도 회전 버튼
│  [🗑 삭제]   │  ← 삭제 버튼
└─────────────┘
```

**순서 변경**:
- 드래그 앤 드롭으로 순서 변경
- 순서 번호 자동 업데이트
- 실시간 반영

**대표 썸네일 선택**:
- 라디오 버튼으로 하나만 선택 가능
- 선택한 이미지가 1:1 비율로 자동 크롭됨
- 크롭 프리뷰 표시 (선택 사항)

**이미지 회전**:
- 90도씩 회전 (시계방향)
- 즉시 프리뷰 업데이트
- 실제 파일도 회전 (서버에 저장)

**이미지 삭제**:
```
[삭제] 버튼 클릭
   ↓
"정말 삭제하시겠습니까?" 확인 모달
   ↓
확인 → 삭제
취소 → 그대로
```

##### 4.3 이미지 캡션 입력

**캡션 에디터**:
- 이미지별로 개별 에디터
- WYSIWYG 스타일 (Tiptap)
- 최대 1000자

**에디터 툴바**:
```
[B] [I] [U] [🔗 작업 링크]
```
- **B**: Bold (진하게)
- **I**: Italic (기울임)
- **U**: Underline (밑줄)
- **🔗**: 작업 링크 삽입

**작업 링크 삽입 플로우**:
```
1. 캡션에서 텍스트 드래그 선택
   예: "아름다운 바다"
   
2. [🔗 작업 링크] 버튼 클릭
   
3. 모달 팝업
   ┌────────────────────────────┐
   │ 작업 검색                  │
   │ [검색: _____________]      │
   │                            │
   │ 검색 결과:                 │
   │  [썸네일] 아름다운 바다     │
   │  [썸네일] 아름다운 하늘     │
   │  [썸네일] 푸른 바다        │
   │                            │
   │      [취소]  [선택]        │
   └────────────────────────────┘
   
4. 작업 선택
   
5. 자동으로 HTML 링크 생성
   <a href="/works/work-123" data-work-id="work-123">
     아름다운 바다
   </a>
   
6. 캡션에 삽입
```

**작업 링크 표시**:
- 에디터 내에서 링크는 파란색 + 밑줄
- 호버 시 툴팁으로 작업명 표시
- 클릭 시 링크 수정 가능

##### 4.4 카테고리 선택

**문장형 카테고리**:
```
[✓] 물 (물은 아름다운 불과 같다)
[ ] 아름다운 (물은 아름다운 불과 같다)
[✓] 불 (물은 아름다운 불과 같다)
```
- 체크박스로 다중 선택
- 괄호 안에 전체 문장 표시
- 선택한 카테고리는 작업과 연결됨

**텍스트형 카테고리**:
```
[✓] 디자인
[ ] 조각
[✓] 추상
[✓] 설치미술
```
- 체크박스로 다중 선택
- 새 카테고리 추가 버튼 (하단)

**새 카테고리 추가**:
```
[+ 새 카테고리] 버튼 클릭
   ↓
작은 입력창 표시
[새 카테고리명: ____________] [추가]
   ↓
즉시 목록에 추가되고 체크됨
```

##### 4.5 저장 및 상태 관리

**임시 저장**:
- [임시저장] 버튼
- 현재 입력한 내용을 서버에 저장 (비공개 상태)
- 성공 시 토스트: "임시 저장되었습니다"
- 언제든 [작업 수정]으로 돌아와서 이어서 작업 가능

**미리보기**:
- [미리보기] 버튼
- 새 탭에서 실제 사용자 페이지 렌더링
- 저장하지 않은 변경사항도 미리보기 가능 (임시 URL 생성)

**저장**:
- [저장] 버튼
- 필수 항목 검증:
  - 제목 (필수)
  - 상세 설명 (필수)
  - 이미지 최소 1장
  - 대표 썸네일 선택
- 검증 실패 시:
  - 해당 필드로 스크롤
  - 에러 메시지 표시
- 검증 성공 시:
  - 서버에 저장
  - 작업 목록으로 이동
  - 성공 토스트 표시

**취소**:
- [취소] 버튼
- 변경사항이 있으면 확인 모달:
  ```
  "저장하지 않은 변경사항이 있습니다.
   정말 나가시겠습니까?"
   [임시저장] [나가기] [계속 작업]
  ```

**자동 저장** (선택 사항):
- 5분마다 자동으로 임시 저장
- 화면 상단에 "자동 저장됨 (5분 전)" 표시

```typescript
    };
```

## 🎯 핵심 사용자 시나리오

### 시나리오 1: 전시 끝나고 작품 일괄 업로드 (데스크탑)

**목표**: 전시에서 찍은 작품 사진 50장을 빠르게 업로드

**단계**:
1. 관리자 페이지 로그인
2. [+ 새 작업] 클릭
3. 제목 입력: "2024 졸업 전시 - 작품 1"
4. 이미지 업로드 영역에 10장 드래그 앤 드롭
5. 업로드 중... (진행률 표시)
6. 대표 썸네일 선택 (첫 번째 이미지)
7. [임시저장] → 다음 작품으로
8. 10개 작품 반복 업로드
9. 나중에 카테고리와 캡션 추가

**소요 시간**: 작품 1개당 2-3분 (10개 작품 약 30분)

---

### 시나리오 2: 공모전 지원 전 작품 순서 조정 (데스크탑)

**목표**: 대표 작품들을 앞으로 배치

**단계**:
1. [카테고리 관리] → [텍스트형 카테고리]
2. "대표작" 카테고리 선택
3. [순서 변경] 클릭
4. 드래그로 작품 순서 조정
   - 가장 잘 나온 작품을 맨 앞으로
   - 시리즈물은 연속으로 배치
5. [저장]
6. [미리보기]로 확인

**소요 시간**: 5분

---

### 시나리오 3: 작품에 캡션 추가하며 다른 작품 링크 걸기 (데스크탑)

**목표**: 작품 설명에 관련 작품 링크 추가

**단계**:
1. [작업 관리] → 수정할 작품 선택
2. 이미지 2번 캡션 입력:
   "이 작품은 <드래그>아름다운 바다</드래그> 시리즈에서 영감을 받았습니다"
3. "아름다운 바다" 텍스트 선택 상태에서
4. [🔗 작업 링크] 버튼 클릭
5. 검색창에 "바다" 입력
6. 검색 결과에서 "아름다운 바다" 작품 선택
7. 자동으로 링크 생성됨
8. [저장]
9. [미리보기]에서 링크 hover 동작 확인

**소요 시간**: 2분

---

### 시나리오 4: 🆕 외출 중 급하게 작품 정보 수정 (모바일)

**목표**: 카페에서 스마트폰으로 작품 공개 상태 변경 및 설명 수정

**단계**:
1. 모바일 브라우저에서 관리자 페이지 접속
2. Google 로그인 (자동 로그인 유지)
3. 햄버거 메뉴 → [작업 관리]
4. 작품 목록에서 해당 작품 카드 탭
5. 수정 화면 진입
6. 공개 상태 토글 (비공개 → 공개)
7. 설명 텍스트 수정 (모바일 키보드 사용)
8. 하단 [저장] 버튼 탭
9. 성공 메시지 확인

**소요 시간**: 1-2분

---

### 시나리오 5: 🆕 전시 현장에서 작품 사진 바로 업로드 (모바일)

**목표**: 전시 현장에서 스마트폰으로 촬영한 사진을 즉시 업로드

**단계**:
1. 스마트폰으로 작품 사진 5장 촬영
2. 관리자 페이지 접속 (PWA 앱으로 저장해두면 더 빠름)
3. 하단 FAB(+) 버튼 탭
4. 제목 입력: "전시 현장 - 작품명"
5. [이미지 추가] → [카메라로 촬영] 또는 [갤러리에서 선택]
6. 방금 찍은 5장 선택

### 작업 목록 성능 최적화 (핵심!)

작업 목록은 사용자가 가장 자주 접근하는 화면이므로 최우선 최적화 대상

#### 1. 가상 스크롤 (Virtual Scrolling)
- **목적**: 100개 이상 작업이 있어도 빠른 렌더링
- **방법**: 
  - react-window 또는 Ant Design Table의 virtual scroll 활용
  - 화면에 보이는 10-20개만 실제 렌더링
  - 스크롤 시 동적으로 렌더링
- **효과**: 1000개 작업도 초기 렌더링 1초 이내

#### 2. 이미지 최적화
- **썸네일 사이즈**: 최대 100x100px (실제 표시 크기)
- **Lazy Loading**: 
  - 화면에 보이는 썸네일만 로드
  - IntersectionObserver 활용
  - Ant Design Image 컴포넌트의 lazy 속성 사용
- **WebP 포맷**: 브라우저 지원 시 WebP 우선 로드
- **Placeholder**: 
  - 로딩 중 skeleton 표시
  - 에러 시 fallback 이미지

#### 3. 데이터 페치 최적화
- **Pagination**: 
  - 기본 20개씩 로드
  - 무한 스크롤 또는 페이지네이션
- **캐싱**: 
  - React Query로 5분간 캐싱
  - 이미 본 페이지는 즉시 표시
- **Prefetching**: 
  - 다음 페이지 미리 로드
  - 마우스 호버 시 상세 정보 prefetch

#### 4. 메모이제이션
```typescript
// 컴포넌트 메모이제이션
const WorkCard = React.memo(({ work }) => {
  // 작업 카드 렌더링
});

// 무거운 계산 캐싱
const filteredWorks = useMemo(() => {
  return works.filter(work => matchesFilter(work));
}, [works, filterCriteria]);
```

#### 5. 렌더링 최적화
- **Debouncing**: 검색 입력 500ms 디바운스
- **Throttling**: 스크롤 이벤트 16ms (60fps) 쓰로틀
- **Code Splitting**: 
  - 작업 폼은 lazy import
  - 카테고리 관리는 lazy import
- **Bundle Size**: 
  - 작업 목록 페이지 JS < 200KB (gzipped)
  - Tree shaking으로 불필요한 코드 제거

#### 6. 상태 관리 최적화
- **지역 상태 우선**: 
  - 체크박스 선택은 지역 상태
  - 전역 상태는 최소화
- **선택적 리렌더링**: 
  - 한 작업의 상태 변경이 다른 작업에 영향 없도록
  - Context 분리 (auth, works, ui)

#### 7. 네트워크 최적화
- **GraphQL 또는 필드 선택**: 
  - 목록에서는 필요한 필드만 (id, title, thumbnail, status)
  - 상세 정보는 클릭 시 로드
- **압축**: gzip 또는 brotli 압축
- **CDN**: 썸네일 이미지는 CDN에서 제공
- **HTTP/2**: 다중 연결로 병렬 로드

#### 8. 모바일 최적화
- **터치 반응**: 터치 이벤트 16ms 이내 반응
- **이미지 해상도**: 
  - 모바일은 작은 해상도 썸네일 (50x50px)
  - Retina 디스플레이 대응 (2x)
- **오프라인 대응**: 
  - Service Worker로 목록 캐싱
  - 네트워크 없어도 마지막 상태 표시

- 커스텀 훅

### 통합 테스트

### E2E 테스트 (Playwright)

**주요 시나리오**:
1. 로그인 → 작업 생성 → 저장 → 확인
2. 이미지 업로드 → 순서 변경 → 썸네일 선택
3. 카테고리 생성 → 작업 연결 → 순서 조정

### 사용자 테스트

**테스트 대상**: 실제 미대생 3-5명

**테스트 시나리오**:
1. "작품 5개를 업로드하고 카테고리를 지정해주세요" (데스크탑)
2. "작품 순서를 바꿔주세요" (데스크탑)
3. "캡션에 다른 작품 링크를 추가해주세요" (데스크탑)
4. 🆕 **"모바일로 작품 1개를 업로드해주세요"** (모바일)
5. 🆕 **"모바일에서 작품 공개 상태를 변경해주세요"** (모바일)

**성공 기준**:
- 80% 이상이 도움 없이 완료
- 평균 완료 시간:
  - 데스크탑 작업: < 15분
  - 모바일 작업: < 5분
- 사용자 만족도 4/5 이상
- 🆕 **모바일 사용성 만족도 4/5 이상**

## 🔮 향후 계획 (Phase 4+)

### 우선순위 높음

#### 1. **분석 기능** 📊
**목적**: 어떤 작품이 인기 있는지 파악하여 포트폴리오 개선

**기능**:
- **작품별 조회수**
  - 일별/주별/월별 조회수 그래프
  - 가장 많이 본 작품 TOP 10
  - 평균 체류 시간
  
- **카테고리별 인기도**
  - 카테고리별 클릭 수
  - 어떤 카테고리로 들어오는지 분석
  
- **방문자 통계**
  - 일별 방문자 수
  - 신규 vs 재방문자
  - 접속 시간대 분석
  
- **유입 경로**
  - 어디서 링크 타고 왔는지 (Referrer)
  - 직접 접속 vs 링크 공유
  
- **디바이스 분석**
  - 모바일 vs 데스크탑 비율
  - 브라우저 종류

**구현 방법**:
- Google Analytics 4 연동 (가장 간단)
- 또는 자체 분석 시스템 (Privacy 중시 시)
- 관리자 페이지에 대시보드 추가

**개발 기간**: 1-2주

---

#### 2. **SEO 최적화** 🔍
**목적**: 구글 검색에 작품이 노출되어 더 많은 사람이 볼 수 있도록

**기능**:
- **메타 태그 자동 생성**
  - 작품 제목 → `<title>` 태그
  - 간단한 설명 → `<meta description>`
  - 대표 이미지 → Open Graph 이미지
  
- **Open Graph / Twitter Card**
  - SNS 공유 시 예쁜 카드 표시
  - 작품 썸네일 + 제목 + 설명
  
- **구조화된 데이터 (Schema.org)**
  - CreativeWork 마크업
  - 작가 정보
  - 작품 정보
  
- **Sitemap 자동 생성**
  - XML Sitemap
  - 새 작품 추가 시 자동 업데이트
  - Google Search Console 제출
  
- **이미지 최적화**
  - Alt 태그 자동 생성 (작품 제목 기반)
  - 이미지 파일명 의미있게 (work-title-1.jpg)
  
- **URL 구조**
  - `/works/[작품-제목-slug]` (한글 URL 또는 영문 slug)
  - Clean URL, 의미있는 경로

**구현 방법**:
- Next.js의 경우 `next-seo` 라이브러리
- 작업 저장 시 자동으로 SEO 데이터 생성
- Sitemap은 빌드 시 자동 생성

**개발 기간**: 1주
