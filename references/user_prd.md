# 포트폴리오 사용자 페이지 PRD

## 📋 문서 정보

- **문서 버전**: 1.7
- **작성일**: 2025년 11월 2일
- **최종 수정일**: 2025년 12월 19일 (v1.7 - 영상(YouTube Embed) 지원 추가)
- **대상**: 포트폴리오 방문자 (작품 감상자)
- **목적**: 작품을 아름답게 전시하고, 직관적으로 탐색할 수 있는 갤러리 경험 제공

### 변경 이력

**v1.1 (2025년 1월)**
- 호버 시 좌→우로 흐르는 bold 애니메이션 구현 (500ms duration)
- 점(˙) 표시 크기: 14px로 변경
- 상단 선택 영역 제거 (선택된 카테고리는 원래 위치에서만 표시)
- 텍스트형 카테고리에도 좌→우 bold 애니메이션 적용
- 애니메이션 구현 코드 예시 업데이트

**v1.2 (2025년 1월 - 현재 구현 반영)**
- 가로 스크롤 리스트에 fading edge 및 화살표 네비게이션 추가
- 모든 스크롤바 완전히 투명 처리 (미니멀 디자인)
- 작품 상세 페이지에서 같은 페이지 내 작품 선택 기능 (페이지 이동 없음)
- 작품 제목 리스트를 가로 스크롤 리스트로 변경 (sticky header 지원)
- 하나의 작업이 여러 카테고리를 가질 수 있는 규칙 명시
- Apple/Google 스타일의 부드러운 애니메이션 적용 (cubic-bezier easing)

**v1.3 (2025년 11월 2일 - Floating Window 및 애니메이션 개선)**
- 작업 링크 호버 시 Floating Window 구현 완료 (위키피디아 스타일)
  - 링크 hover 후 500ms 지연 후 프리뷰 표시
  - 링크 바로 아래 중앙에 고정 위치 배치 (마우스 이동에 따라 변경되지 않음)
  - 안전 마진 50px 적용: 링크/Floating Window 주변 50px 내에서는 프리뷰 유지
  - 마우스가 안전 마진 밖으로 나가면 프리뷰 사라짐
  - 위키피디아 스타일의 자연스러운 애니메이션 (fade-in/out)
- 작업 목록 hover 애니메이션 개선
  - 고정 레이아웃 적용: 제목/설명 영역 최소 높이 고정으로 떨림 현상 제거
  - 설명 표시/숨김 시 opacity 전환 사용 (height 애니메이션 제거)
  - 마진 고정으로 레이아웃 shift 방지
  - transform-origin 중앙 설정으로 이미지 확대 시 자연스러운 효과
- Floating Window 재사용성 개선
  - 한 번 hover 후 다시 나타나지 않던 문제 해결
  - 다른 링크로 이동 시 이전 프리뷰 즉시 닫기
  - 스크롤 시 Floating Window 자동 숨김

**v1.4 (2025년 12월 8일 - 전시명 카테고리 및 디자인 반영)**
- 카테고리 구조 변경
  - 텍스트형 카테고리 → 전시명 카테고리로 명칭 및 구조 변경
  - 전시명 카테고리 표시 형식: `<작업명>` + 전시유형, 공간, 년도
  - 전시명 카테고리는 통으로 클릭 (문장형과 달리 키워드별 분리 없음)
- 작업 리스트 페이지 디자인 변경
  - 작업 카드 레이아웃: 제목 + 이미지 아이콘 조합으로 변경
  - 제목 형식: 「'작품명'」, 년도
- 카테고리별 작업 리스트 정렬 방향 추가
  - 문장형 카테고리: 좌측 정렬, 좌 → 우 방향
  - 전시명 카테고리: 우측 정렬, 우 → 좌 방향
- 이미지 호버 효과 변경
  - 기존 확대 효과 → 붉은 프레임(border: 2px solid red)으로 변경
- 캡션 구조 변경
  - 이미지별 캡션 → 작업당 1개의 캡션으로 변경
  - 캡션 위치: 대표이미지 옆 고정 (이미지 스크롤과 별개로 위치 고정)
- 캡션 내 작업 링크 스타일 및 인터랙션
  - 캡션 내 포함된 작업: 진한 명도로 표시 (문장형 카테고리 disabled 표현과 유사)
  - hover: 500ms 후 프리뷰 표시 (Floating Window)
  - click: 모달로 작업 상세 표시 (제2의 상세 페이지)
- 모달 상세 페이지 구조 추가
  - 레이아웃: 상단(작업명) + 좌측(이미지 세로 나열) + 우측(캡션 고정)
  - 모달 내 캡션에도 작업 링크 삽입 가능
  - 모달 내 작업 링크 클릭 시: 현재 모달 닫힘 → 새 모달로 해당 작업 표시
  - 재귀적으로 다른 작업으로 이동 가능
- 이미지 스크롤바 타임라인 UI 추가
  - 화면 좌측에 위치, 점 --- 세로선 --- 점 형식
  - 점은 각 이미지의 좌측 상단 코너에 정렬
  - Snap 효과 및 Interpolation 애니메이션
- 작업 선택 시 리스트 상태 변화 추가
  - 카테고리 클릭: 작업명 + 썸네일 리스트 표시
  - 작업 상세 화면 진입: 썸네일 사라지고 작업명만 남음
- 카테고리 섹션 높이 제한 추가
  - 화면 세로 길이의 25% 초과 시 fading + 스크롤 영역으로 전환
  - 좌/우 각각 독립적 적용
- 전체 레이아웃 와이어프레임 업데이트
  - PDF 디자인 시안 반영
  - 실제 예시 텍스트 적용

**v1.7 (2025년 12월 19일 - 영상(YouTube Embed) 지원 추가)**
- 영상(YouTube Embed) 지원 추가
  - 작업 상세 페이지에서 이미지와 함께 YouTube 영상 표시 가능
  - 이미지와 영상이 하나의 미디어 목록으로 통합 관리됨
  - 관리자가 설정한 순서대로 이미지/영상이 순차적으로 표시
- 미디어 타임라인 UI 업데이트
  - 영상 아이템에 재생 아이콘(▶️) 표시로 이미지와 구분
  - 타임라인 점이 영상 위치에서도 정상 동작
- 모달 내 영상 표시 지원
  - 모달 상세 페이지에서도 이미지와 영상 함께 표시
  - 영상은 YouTube embed iframe으로 렌더링
- 데이터 구조 추가: WorkVideo, MediaItem 타입
- 반응형 영상 embed: 16:9 비율 유지, 모바일에서도 최적화

**v1.6 (2025년 12월 18일 - 모달/상세페이지 UX 개선)**
- 모달 내 캡션 작업 링크 인터랙션 개선
  - 모달 내에서도 팝업 프리뷰 표시 (작업 상세 화면과 동일)
  - 팝업 스타일: 회색 배경, fade out 테두리, 부드러운 모션 애니메이션
  - 팝업 클릭 시 새 모달로 확대 (가로세로 커지는 애니메이션)
- 상세페이지 캡션 위치 조정: 오른쪽 아래로 약 10% 이동
- 상세페이지 좌측 스크롤바 단순화: 고정 점선 + 점 이동 애니메이션으로 변경
- 상세페이지 작업 리스트 호버 시 썸네일 재표시 + 붉은 프레임 처리
- 상세페이지에서 다른 카테고리 클릭 시 썸네일 리스트로 연결 (첫 작업 상세가 아닌)
- 작업 리스트 스크롤 인디케이터 구분: 텍스트 옆 `...`, 썸네일 옆 `>>` / `<<`
- 전시명 카테고리 호버/액티브 상태 개선: 전시 제목(`<작업명>`)만 진한 명도 적용, 간단 설명(전시유형, 공간, 년도)은 기본 색상 유지

**v1.5 (2025년 12월 8일 - 구현 완료 및 세부 조정)**
- 작업 카드 컴포넌트 통합
  - 홈/상세 페이지에서 동일한 `WorkTitleButton` 컴포넌트 재사용
  - `showThumbnail` prop으로 썸네일 표시 여부 제어
  - 홈: 썸네일 표시 (80x80px) + 제목 + 년도
  - 상세: 제목 + 년도만 표시 (썸네일 없음)
- 작업 카드 스타일 세부 조정
  - 썸네일 크기: 80px × 80px (PRD 스펙 준수)
  - 카드 간 간격: 32px (PRD 스펙 준수)
  - 제목 폰트: 12px
  - 년도 폰트: 10px (color-text-secondary)
  - 호버 효과: 제목 bold + 썸네일 빨간 테두리 (border: 2px solid red)
- 타임라인 UI 개선
  - 점선 스타일: `linear-gradient` 기반 점선 (backgroundSize: 1px 6px)
  - 동적 선 길이: 활성 이미지 기준으로 선 길이 변화 (100px/60px/30px)
  - 이미지 1개일 때 타임라인 자동 숨김
  - Intersection Observer로 현재 이미지 감지 (중앙 기반 점수 계산)
- z-index 이슈 해결
  - 카테고리 영역 z-index: 100 (main 영역 위에 표시)
  - 클릭 이벤트 정상 동작 확인
- 호버 스타일 전환 개선
  - WebkitTextStroke: '0px transparent' → '1px var(--color-category-hover-stroke)'
  - transition에 `-webkit-text-stroke` 추가로 부드러운 전환

## 🎯 프로젝트 목표

### 핵심 가치
"여백의 미를 살린 미니멀한 디지털 갤러리"

### 주요 목표
1. **여백의 미**: 불필요한 요소 제거, 작품에 집중
2. **직관적 탐색**: 카테고리 중심의 자연스러운 작품 발견
3. **완전한 반응형**: 모바일부터 대형 모니터까지 아름다운 경험
4. **빠른 로딩**: 이미지 최적화로 < 2초 로딩
5. **몰입감**: 작품 감상에 방해되지 않는 인터페이스

- **타이포그래피 중심**: 텍스트 가독성 최우선

### 레이아웃 그리드

```css
/* 컨테이너 */
--container-max-width: 1920px;
--container-padding: var(--space-6);   /* 48px */

/* 좌/우 카테고리 영역 (고정 박스 없음) */
--category-margin-left: var(--space-6);    /* 48px - 좌측 시작점 */
--category-margin-right: var(--space-6);   /* 48px - 우측 끝점 */
--category-spacing: var(--space-5);        /* 40px - 문장 간 간격 */
--category-font-size: 20px;
--category-line-height: 1.8;

/* 중앙 컨텐츠 영역 */
--content-gap: var(--space-8);         /* 카테고리와 중앙 간격 */

/* 점(˙) 표시 */
--dot-size: 4px;
--dot-offset-top: -8px;                /* 글자 정중앙 위 */
--dot-color: var(--color-black);
/* 참고: 실제 구현에서는 점의 font-size가 14px로 설정됨 */
```

- . . . 표시: 스크롤 가능 여부에 따라 표시
- 스크롤바: 완전히 투명 처리
- 마우스 드래그/휠로도 가로 스크롤 가능

**작업 선택 시 리스트 상태 변화**:
```

        2023              2023               2023

    「'종이별'」      「'종이학 다리'」   「'종이학'」
        2023              2023               2023

            ↓ 다른 작업에 마우스 호버

        2023              2023               2023
```

- **카테고리 클릭 시**: 작업명 + 썸네일 리스트 표시
- **작업 클릭 후 (상세 화면 진입)**: 썸네일 사라지고 **작업명만 남음**
- **선택된 작업**: bold 또는 underline으로 표시
- **전환 애니메이션**: 썸네일 fade-out (0.2s ease-out)
- **🆕 상세 화면에서 다른 작업 호버 시**:
  - 호버된 작업의 **썸네일이 다시 표시됨** (fade-in 0.2s)
  - 썸네일에 **붉은 프레임 처리** (border: 2px solid red)
  - 호버 해제 시 썸네일 다시 사라짐 (fade-out 0.2s)
  - 현재 선택된 작업(상세 보는 중)은 호버해도 썸네일 표시 안 함

**⚠️ 카테고리 섹션 유지**:
- 작업 상세 화면에 진입해도 **상단 카테고리 섹션은 사라지지 않음**
- 좌측 문장형 카테고리, 우측 전시명 카테고리 모두 계속 표시
- 사용자는 언제든지 다른 카테고리를 선택하여 이동 가능
- 현재 선택된 카테고리는 active 상태 유지 (점 표시 또는 bold)

**🆕 상세 화면에서 다른 카테고리 클릭 시 동작**:
- **기존 동작 (변경 전)**: 새 카테고리 내 첫 번째 작업의 상세 페이지로 바로 연결
- **변경된 동작**: 새 카테고리의 **작품 썸네일 리스트**로 연결
  - 상세 화면이 닫히고 홈 화면의 리스트 상태로 전환
  - 새 카테고리가 active 상태로 표시 (점 또는 bold)
  - 해당 카테고리의 모든 작업이 썸네일과 함께 표시됨
  - 사용자가 원하는 작업을 선택하여 상세 화면에 진입
- **UX 목적**: 카테고리 전환 시 사용자에게 선택권 제공, 자연스러운 탐색 흐름 유지

**애니메이션**:
- 작품 카드 등장: fade-in + scale (0.95 → 1) + translateY (8px → 0)
- Duration: 0.35s
- Easing: cubic-bezier(0.25, 0.1, 0.25, 1) - Apple 스타일
- Stagger: 0.03s (순차적 등장)

#### 2.2 작품 제목 리스트 (가로 스크롤)

**작품 제목 리스트**:
```
[←] 작품1  작품2  작품3  작품4  작품5  ... [→]
    (현재 작품 bold)
```

**기능**:
- 현재 보고 있는 작품: **Bold** 처리
- 다른 작품 클릭 시: **페이지 이동 없이** 같은 페이지에서 선택된 작품만 bold 처리하고 해당 작품의 이미지 표시
- 가로 스크롤 리스트: 한 줄로 나열
- Sticky header: 스크롤 시 Header와 같은 높이로 이동
  - 초기 상태: Header 아래 60px
  - 스크롤 후: 최상단 (Header와 같은 줄)

**스타일**:
- 폰트: 10px
- 간격: 카드 간 16px
- 호버: underline
- 현재 작품: bold
- 배경: transparent (항상)
- 위치: 화면 너비의 60%, 최대 600px, 중앙 정렬

**스크롤 네비게이션**:
- 좌우 화살표: 컨테이너 외부 좌우 끝에 배치 (-40px)
- Fading edge: 스크롤 위치에 따라 동적 표시/숨김
- 스크롤바: 완전히 투명

#### 2.3 작품 미디어 및 캡션

**레이아웃**:
- 좌측: 이미지와 영상을 세로로 나열 (스크롤 가능)
- 우측: 캡션 박스 (위치 고정, 스크롤과 별개로 고정)
- 이미지 원본 비율 유지
- 영상: 16:9 비율 고정 (반응형)
- 미디어 간 간격: 80px
- **미디어 순서**: 관리자가 설정한 `order` 값 기준으로 이미지와 영상 함께 정렬

**영상(YouTube Embed) 표시**:
```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│    ┌───────────────────────────────────┐                    │
│    │                                   │                    │
│    │      ▶️ YouTube Embed              │                    │
│    │      (16:9 비율)                  │                    │
│    │                                   │                    │
│    └───────────────────────────────────┘                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
- **렌더링**: YouTube embed iframe 사용
- **비율**: 16:9 고정 (padding-top: 56.25% 기법)
- **반응형**: 컨테이너 너비에 맞게 자동 조절
- **제어**: 재생/일시정지, 볼륨, 전체화면 (YouTube 기본 컨트롤)
- **자동재생 없음**: 사용자가 직접 재생 버튼 클릭 필요

**미디어 스크롤바 (단순화된 타임라인 UI)**:
```
  ·         ← 고정 점선 (시작)
  │
  │
  ●────┐    ← 현재 위치를 나타내는 이동 점 (●)
  │    │  [이미지 1]
  │    │
  ·    │    ← 고정 점선
  │    │  [▶️ 영상 1]        ← 영상 아이템 (재생 아이콘으로 구분)
  │    │
  ·    │    ← 고정 점선
  │    │  [이미지 2]
  │    │
  ·    │    ← 고정 점선
  │    │  [이미지 3]
  │    │
  ·         ← 고정 점선 (끝)
```
- **위치**: 화면 좌측 (미디어 영역의 왼쪽)
- **디자인 (단순화)**:
  - **고정 점선**: 세로 점선(·)이 전체 스크롤 영역에 걸쳐 고정 표시
  - **이동 점**: 현재 보이는 미디어 위치에 따라 점(●)이 점선을 따라 이동
  - 복잡한 타임라인 제거, 단순한 점선 + 이동 점 구조로 변경
  - **이미지와 영상 동일 처리**: 타임라인은 미디어 타입을 구분하지 않음
- **점선 스타일**: 연한 회색 (gray-300), dashed line
- **이동 점 스타일**: 진한 검정 (black), 크기 6px
- **점 이동 애니메이션**: 스크롤에 따라 부드럽게 이동 (ease-out, 0.2s)
- **클릭 불가**: 점선/점 클릭 시 동작 없음 (단순 표시용)

**스크롤 동작**:
- **Snap 효과**: 스크롤 시 미디어 단위로 정렬 (scroll-snap-type: y mandatory)
- **점 이동**: 스크롤 위치에 따라 이동 점(●)이 고정 점선을 따라 아래로 이동
- **Interpolation**: 점 이동 시 부드러운 애니메이션 (ease-out)
- **영상 처리**: 영상 아이템도 이미지와 동일하게 스냅 대상

**미디어 + 캡션 레이아웃**:

      ```
      └────────────────────────────┘ ← fade out 테두리 (부드러운 경계)
      ```

      - **팝업 스타일**:
        - 배경: 회색 (gray-200 또는 gray-300)
        - 모양: 사각형
        - 테두리: **fade out 처리** (box-shadow blur로 부드러운 경계 연출)

      - **표시 조건**: 링크 hover 후 500ms 지연 후 표시
      - **위치**: 링크 바로 아래 또는 옆에 고정 배치
      - **애니메이션**: fade-in (0.2s ease-out)

   2. **click 시 팝업 → 모달 전환**:
      ```
      [팝업 상태]                    [모달로 확대]
      ┌──────────┐                 ┌─────────────────────┐
      │ 작업명    │   ──────→      │                     │
      ```

      - **전환 애니메이션**:
        - 팝업에서 모달로 **가로세로가 커지는** Motion animation
        - Duration: 0.3s
        - Easing: cubic-bezier(0.25, 0.1, 0.25, 1) - Apple 스타일
        - **과하지 않은** 자연스러운 확대 효과
        - transform: scale(0.4 → 1) + opacity (0.8 → 1)
      - **모달 배경**: 팝업과 **동일한 회색** 사용 (일관성 유지)
      - 현재 열린 모달이 닫히고 새 모달 열림

## 🎭 인터랙션 및 애니메이션

### 기본 원칙
- **부드럽게**: 모든 전환은 Apple/Google 스타일 easing 곡선 사용
- **빠르게**: 애니메이션 시간 0.2-0.4초
- **의미있게**: 필요한 곳에만 애니메이션
- **Easing 함수**: 
  - Material Design 표준: `cubic-bezier(0.4, 0, 0.2, 1)`
  - Apple 스타일: `cubic-bezier(0.25, 0.1, 0.25, 1)`

### 주요 애니메이션

#### 1. 페이지 로드
```
1. 헤더 페이드인 (0.3s)
2. 좌측 카테고리 영역 페이드인 (0.4s) - absolute 배치로 자유롭게 표시
3. 우측 카테고리 영역 페이드인 (0.4s) - absolute 배치로 자유롭게 표시
4. 중앙 안내 문구 페이드인 (0.5s)
```

#### 2. 카테고리 선택 애니메이션

**문장형 카테고리 (좌측) 선택 시**:
```
[Step 1] 클릭 (0~0.2s)
물과 바람은 하나가 되어서  ← 원래 위치에서 바로 선택 상태로 전환

[Step 2] 점 표시 (0.4~0.7s)
     ˙                      ← 선택된 단어 위에 점(˙) 페이드인 (font-size: 14px)
물과 바람은 하나가 되어서  ← 원래 위치 유지, bold 상태 유지

[Step 3] 작품 카드 표시 (0.6~1.0s)
중앙 영역에 작품 카드들:
- 페이드인 + 위에서 아래로 (translateY)
- Stagger: 0.05s (순차적으로)

참고: 상단 영역에 선택된 카테고리 표시하지 않음 (PRD 변경사항)
```

**텍스트형 카테고리 (우측) 선택 시**:
```
[Step 1] 클릭 (0~0.2s)
┌──────────────┐
│  디자인      │  ← 선택된 항목 하이라이트 (배경 gray-100)
│  조각        │
└──────────────┘

[Step 2] 작품 카드 표시 (0.2~0.6s)
중앙 영역에 작품 카드들:
- 페이드인 + 위에서 아래로
- Stagger: 0.05s
```

## 📝 디자인 시스템 스켈레톤

### 컴포넌트 목록

#### 기본 컴포넌트
```typescript
// Typography
<H1>제목</H1>
<H2>부제목</H2>
<Body>본문</Body>
<Caption>캡션</Caption>

// Layout
<Container>
<Section>
<Grid cols={8}>
<Flex>

// Interactive
<Button variant="primary">버튼</Button>
<Link href="/">링크</Link>
```

#### 작품 관련 컴포넌트
```typescript
<WorkCard
  thumbnail="/image.jpg"
  title="작품명"
  onClick={() => {}}
/>

<WorkGrid
  works={works}
  columns={8}
/>

<WorkDetail
  work={work}
  category="물"
/>

<WorkImage
  src="/image.jpg"
  caption="캡션 텍스트"
/>

<WorkVideo
  youtubeVideoId="xxx"
  title="영상 제목"
/>

// 통합 미디어 리스트 (이미지 + 영상)
<MediaList
  items={[
    { type: 'image', data: imageData },
    { type: 'video', data: videoData },
  ]}
  onMediaClick={(item) => {}}
/>
```

#### 카테고리 컴포넌트
```typescript
// 좌측 영역 - 문장형 카테고리 (고정 박스 없음, 자유롭게 배치)
<SentenceCategoryArea
  sentences={[
    {
      text: "물과 바람은 하나가 되어서",
      keywords: [
        { text: "물", startIndex: 0, endIndex: 1 },
        { text: "바람", startIndex: 3, endIndex: 5 }
      ]

    {
      text: "고요함 속에 소리가 있다",

        { text: "고요함", startIndex: 0, endIndex: 3 },

// 우측 영역 - 텍스트형 카테고리 (고정 박스 없음, 우측 정렬)
```
