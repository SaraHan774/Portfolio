# 포트폴리오 사용자 페이지 PRD

## 📋 문서 정보

- **문서 버전**: 1.3
- **작성일**: 2025년 11월 2일
- **최종 수정일**: 2025년 11월 2일 (v1.3 - Floating Window 및 애니메이션 개선 반영)
- **대상**: 포트폴리오 방문자 (작품 감상자)
- **목적**: 작품을 아름답게 전시하고, 직관적으로 탐색할 수 있는 갤러리 경험 제공

### 변경 이력

**v1.1 (2025년 1월)**
- 호버 시 좌→우로 흐르는 bold 애니메이션 구현 (500ms duration)
- 점(˙) 표시 크기: 14px로 변경
- 상단 선택 영역 제거 (선택된 카테고리는 원래 위치에서만 표시)
- 텍스트형 카테고리에도 좌→우 bold 애니메이션 적용
- 애니메이션 구현 코드 예시 업데이트

**v1.2 (2025년 1월 - 현재 구현 반영)**
- 가로 스크롤 리스트에 fading edge 및 화살표 네비게이션 추가
- 모든 스크롤바 완전히 투명 처리 (미니멀 디자인)
- 작품 상세 페이지에서 같은 페이지 내 작품 선택 기능 (페이지 이동 없음)
- 작품 제목 리스트를 가로 스크롤 리스트로 변경 (sticky header 지원)
- 하나의 작업이 여러 카테고리를 가질 수 있는 규칙 명시
- Apple/Google 스타일의 부드러운 애니메이션 적용 (cubic-bezier easing)

**v1.3 (2025년 11월 2일 - Floating Window 및 애니메이션 개선)**
- 작업 링크 호버 시 Floating Window 구현 완료 (위키피디아 스타일)
  - 링크 hover 후 500ms 지연 후 프리뷰 표시
  - 링크 바로 아래 중앙에 고정 위치 배치 (마우스 이동에 따라 변경되지 않음)
  - 안전 마진 50px 적용: 링크/Floating Window 주변 50px 내에서는 프리뷰 유지
  - 마우스가 안전 마진 밖으로 나가면 프리뷰 사라짐
  - 위키피디아 스타일의 자연스러운 애니메이션 (fade-in/out)
- 작업 목록 hover 애니메이션 개선
  - 고정 레이아웃 적용: 제목/설명 영역 최소 높이 고정으로 떨림 현상 제거
  - 설명 표시/숨김 시 opacity 전환 사용 (height 애니메이션 제거)
  - 마진 고정으로 레이아웃 shift 방지
  - transform-origin 중앙 설정으로 이미지 확대 시 자연스러운 효과
- Floating Window 재사용성 개선
  - 한 번 hover 후 다시 나타나지 않던 문제 해결
  - 다른 링크로 이동 시 이전 프리뷰 즉시 닫기
  - 스크롤 시 Floating Window 자동 숨김

## 🎯 프로젝트 목표

### 핵심 가치
"여백의 미를 살린 미니멀한 디지털 갤러리"

### 주요 목표
1. **여백의 미**: 불필요한 요소 제거, 작품에 집중
2. **직관적 탐색**: 카테고리 중심의 자연스러운 작품 발견
3. **완전한 반응형**: 모바일부터 대형 모니터까지 아름다운 경험
4. **빠른 로딩**: 이미지 최적화로 < 2초 로딩
5. **몰입감**: 작품 감상에 방해되지 않는 인터페이스

## 🖼️ 화면별 상세 기획

### 1. 메인 페이지 (작품 목록)

#### 1.1 헤더

**데스크탑**:
```
┌─────────────────────────────────────────────────────────┐
│  Portfolio                                      [Home]  │  ← 고정 헤더
└─────────────────────────────────────────────────────────┘
```

**구성 요소**:
- **좌측**: "Portfolio" 텍스트 (클릭 시 홈으로)
- **우측**: [Home] 버튼
- **스타일**: 
  - 배경: 흰색
  - 테두리: 하단에 얇은 회색 선 (1px)
  - 높이: 60px
  - Sticky 고정

**모바일**:
```
┌─────────────────────────┐
│  Portfolio        [☰]  │  ← 햄버거 메뉴
└─────────────────────────┘
```

#### 1.2 좌측 영역 (문장형 카테고리)

**데스크탑**:
```
물과 바람은              ← 호버 시 "물" bold
하나가 되어서            ← 호버 시 "바람" bold

고요함 속에              ← 호버 시 "고요함", "소리" bold
소리가 있다

(박스 없음, 자유롭게 배치)
```

**레이아웃 특징**:
- 고정된 박스/사이드바 없음
- 좌측 여백(48px)에서 시작
- 가로 길이 제약 없음 (문장 길이에 따라 자연스럽게)
- 문장 간 간격: 40px (var(--space-5))
- 노트에 끄적인 느낌

**인터랙션**:

1. **호버 (Hover)**
   - 문장 내 키워드(카테고리)에 마우스 호버 시:
     - 해당 단어가 **좌→우로 순차적으로 Bold 처리** (약 500ms duration)
     - 각 글자마다 순차적으로 bold 적용 (stagger: 50ms 간격)
     - 커서: pointer
     - Transition: 부드러운 ease-in-out 곡선 사용
   
2. **클릭 (Click)**
   ```
   [클릭 전]
       
   물과 바람은 하나가 되어서  ← 좌측에 자유롭게 배치
   
   [클릭 시: "물" 클릭]
        ˙                     ← "물" 위에 점 페이드인 (0.3s)
   물과 바람은 하나가 되어서  ← 원래 위치 유지, 중앙에 작품 표시
   ```

   **애니메이션 상세**:
   - **Step 1** (0~0.3s): 선택된 단어 위에 점(˙) 페이드인
     - 위치: 단어 정중앙 위 (vertical-align)
     - 크기: 14px (font-size)
     - 색상: black
     - `opacity: 0 → 1`, `ease-out`, delay: 0.4s
   - **Step 2** (0.2~0.6s): 중앙 영역에 작품 카드 표시
     - 페이드인 + 위에서 아래로 (translateY)
     - Stagger: 0.05s (순차적으로)

**스타일**:
- 위치: 좌측 여백 48px에서 시작
- 최대 너비: 없음 (자유롭게)
- 폰트: 20px, line-height 1.8
- 색상: 
  - 기본: gray-600
  - 호버: black + bold
  - 선택됨: 원래 위치에서는 기본 상태 유지

**점(˙) 스타일 상세**:
```css
.selected-word {
  position: relative;
}

.selected-word::before {
  content: '˙';
  position: absolute;
  top: -8px;  /* 글자 정중앙 위 */
  left: 50%;
  transform: translateX(-50%);
  font-size: 14px;  /* 최종 구현: 14px */
  color: var(--color-black);
  animation: dotFadeIn 0.3s ease-out 0.4s both;
}

@keyframes dotFadeIn {
  from { opacity: 0; }
  to { opacity: 1; }
```

**모바일**:

- 햄버거 메뉴 클릭 시 슬라이드 인
- 선택 시 Drawer 자동 닫힘 + 상단에 문장 표시

#### 1.3 우측 영역 (텍스트형 카테고리)

```
                        회화

(박스 없음, 우측 정렬로 자유롭게 배치)
```

**레이아웃 특징**:
- 고정된 박스/사이드바 없음
- 우측 여백(48px)에서 끝나도록 우측 정렬
- 가로 길이 제약 없음
- 항목 간 간격: 16px (var(--space-2))
- 노트에 끄적인 느낌

**인터랙션**:
- 각 카테고리는 세로로 배치
- **호버 시**: 텍스트가 **좌→우로 순차적으로 Bold 처리** (약 500ms duration)

**스타일**:
- 위치: 우측 여백 48px에서 끝남 (text-align: right)
- 최대 너비: 없음 (자유롭게)

  - 기본: gray-600

#### 1.4 중앙 컨텐츠 영역 (작업 카드)

**초기 상태 (카테고리 미선택)**:

- 텍스트형 카테고리: 상단 표시 없음

**가로 스크롤 리스트 특징**:

- 크기: 화면 너비의 60%, 최대 600px, 높이 150px
- 스크롤바: 완전히 투명 처리 (보이지 않음)
- 좌우 화살표: 컨테이너 외부 좌우 끝에 배치 (transparent 배경)
- Fading edge: 스크롤 위치에 따라 동적으로 표시/숨김
  - 왼쪽: 시작 위치가 아니면 표시 (좌→우 그라데이션)
  - 오른쪽: 끝 위치가 아니면 표시 (우→좌 그라데이션)
  - 애니메이션: 0.4s cubic-bezier(0.4, 0, 0.2, 1)

**카드 구성**:
- **썸네일**: 1:1 정사각형 (70px × 70px)
- **제목**: 썸네일 아래, 작은 텍스트 (10px)
- **설명**: 호버 시 제목 아래에 표시 (8px, 최대 2줄, 20px 고정 높이)
- **간격**: 카드 간 16px
- **호버**: 
  - 썸네일 약간 확대 (scale 1.02, transform-origin: center)
  - 제목 bold
  - 설명 표시 (opacity 전환, 레이아웃 shift 방지)
  - Transition: 0.2s ease-out
- **레이아웃 고정**: 
  - 제목/설명 영역 최소 높이 36px로 고정 (떨림 방지)
  - 설명 영역 고정 높이 20px (height 애니메이션 제거)
  - 마진 고정으로 레이아웃 shift 완전 차단

**가로 스크롤 기능**:
- 모든 작품을 가로로 나열 (한 줄)
- 좌우 화살표: 컨테이너 외부 좌우 끝에 배치
  - 스크롤 가능할 때만 표시
  - 클릭 시 200px씩 부드럽게 스크롤
  - 배경: transparent
- Fading edge: 스크롤 위치에 따라 동적 표시
  - 그라데이션: white 0% → white 30% → transparent 100%
  - 너비: 40px
  - 전환 애니메이션: 0.4s cubic-bezier(0.4, 0, 0.2, 1)
- 스크롤바: 완전히 투명 처리 (어디서도 보이지 않음)
- 마우스 드래그/휠로도 가로 스크롤 가능

**애니메이션**:
- 작품 카드 등장: fade-in + scale (0.95 → 1) + translateY (8px → 0)
- Duration: 0.35s
- Easing: cubic-bezier(0.25, 0.1, 0.25, 1) - Apple 스타일
- Stagger: 0.03s (순차적 등장)

## 🎭 인터랙션 및 애니메이션

### 기본 원칙
- **부드럽게**: 모든 전환은 Apple/Google 스타일 easing 곡선 사용
- **빠르게**: 애니메이션 시간 0.2-0.4초
- **의미있게**: 필요한 곳에만 애니메이션
- **Easing 함수**: 
  - Material Design 표준: `cubic-bezier(0.4, 0, 0.2, 1)`
  - Apple 스타일: `cubic-bezier(0.25, 0.1, 0.25, 1)`

### 주요 애니메이션

#### 1. 페이지 로드

#### 2. 카테고리 선택 애니메이션

**문장형 카테고리 (좌측) 선택 시**:
```
[Step 1] 클릭 (0~0.2s)
물과 바람은 하나가 되어서  ← 원래 위치에서 바로 선택 상태로 전환

[Step 2] 점 표시 (0.4~0.7s)
     ˙                      ← 선택된 단어 위에 점(˙) 페이드인 (font-size: 14px)
물과 바람은 하나가 되어서  ← 원래 위치 유지, bold 상태 유지

[Step 3] 작품 카드 표시 (0.6~1.0s)
중앙 영역에 작품 카드들:
- 페이드인 + 위에서 아래로 (translateY)
- Stagger: 0.05s (순차적으로)

참고: 상단 영역에 선택된 카테고리 표시하지 않음 (PRD 변경사항)
```
