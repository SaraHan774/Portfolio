# 포트폴리오 사용자 페이지 PRD

## 📋 문서 정보

- **문서 버전**: 1.0
- **작성일**: 2025년 11월 2일
- **대상**: 포트폴리오 방문자 (작품 감상자)
- **목적**: 작품을 아름답게 전시하고, 직관적으로 탐색할 수 있는 갤러리 경험 제공

---

## 🎯 프로젝트 목표

### 핵심 가치
"여백의 미를 살린 미니멀한 디지털 갤러리"

### 주요 목표
1. **여백의 미**: 불필요한 요소 제거, 작품에 집중
2. **직관적 탐색**: 카테고리 중심의 자연스러운 작품 발견
3. **완전한 반응형**: 모바일부터 대형 모니터까지 아름다운 경험
4. **빠른 로딩**: 이미지 최적화로 < 2초 로딩
5. **몰입감**: 작품 감상에 방해되지 않는 인터페이스

## 👤 사용자 페르소나

### 주 사용자: 작품 감상자

**시나리오 1: 공모전 심사위원 (민수, 45세)**
- 포트폴리오 링크를 받아 빠르게 훑어봄
- 특정 카테고리의 작품만 집중해서 봄
- 모바일/태블릿에서 확인
- 니즈: 빠른 로딩, 작품에 집중, 카테고리별 정리

**시나리오 2: 미대생 (지수, 22세)**
- 인스타그램에서 링크 타고 들어옴
- 스마트폰으로 여러 작품 둘러봄
- 관련 작품 링크 따라 계속 탐색
- 니즈: 모바일 친화적, 직관적 네비게이션, 빠른 감상

**시나리오 3: 갤러리 큐레이터 (혜진, 38세)**
- 데스크탑에서 전체 작품 탐색
- 특정 스타일/테마의 작품 찾기
- 작품 간 연결성 파악
- 니즈: 전체 구조 파악, 세련된 UI, 전문성

```
└─────────────────────────┘
```

#### 1.2 좌측 사이드바 (문장형 카테고리)

**데스크탑**:
```
┌──────────────┐
│              │
│  물과        │  ← 호버 시 "물" bold
│  바람은      │  ← 호버 시 "바람" bold
│  하나가      │
│  되어서      │
│              │
│  고요함 속에 │  ← 호버 시 "고요함", "속", "소리" bold
│  소리가 있다 │
│              │
└──────────────┘
```

**인터랙션**:

1. **호버 (Hover)**
   - 문장 내 키워드(카테고리)에 마우스 호버 시:
     - 해당 단어만 **Bold** 처리
     - 커서: pointer
     - Transition: 0.1s ease
   
2. **클릭 (Click)**
   ```
   [클릭 전]
   ┌──────────────┐
   │  물과 바람은  │
   │  하나가      │
   │  되어서      │
   └──────────────┘
   
   [클릭 시: "물" 클릭]
   ┌──────────────┐
   │  물과 바람은  │  ← 문장 전체 잠깐 하이라이트 (0.2s)
   │  하나가      │
   │  되어서      │
   └──────────────┘
        ↓ (애니메이션: 문장이 상단으로 이동)
   
   [헤더 아래 영역]
        ˙
   물과 바람은 하나가 되어서  ← 아래에서 위로 슬라이드 인 (0.4s)
   ```

   **애니메이션 상세**:
   - **Step 1** (0~0.2s): 사이드바 문장 전체 하이라이트 (배경 gray-100)
   - **Step 2** (0.2~0.6s): 상단 영역에 문장 슬라이드 인
     - 시작: `transform: translateY(20px); opacity: 0`
     - 끝: `transform: translateY(0); opacity: 1`
     - Easing: ease-out
   - **Step 3** (0.4~0.6s): 선택된 단어 위에 점(˙) 페이드인
     - 위치: 단어 정중앙 위 (vertical-align)
     - 크기: 4px 원형
     - 색상: black

**스타일**:
- 너비: 240px (고정)
- 패딩: 48px 24px
- 폰트: 20px, line-height 1.8
- 색상: 
  - 기본: gray-600
  - 호버: black + bold
  - 선택됨: 사이드바에서는 기본 상태 유지

**점(˙) 스타일 상세**:
```css
.selected-word {
  position: relative;
}

.selected-word::before {
  content: '˙';
  position: absolute;
  top: -8px;  /* 글자 정중앙 위 */
  left: 50%;
  transform: translateX(-50%);
  font-size: 8px;
  color: var(--color-black);
  animation: dotFadeIn 0.2s ease-out 0.4s both;
}

@keyframes dotFadeIn {
  from { opacity: 0; }
  to { opacity: 1; }
}
```

**모바일**:
- Drawer로 변환
- 햄버거 메뉴 클릭 시 슬라이드 인
- 선택 시 Drawer 자동 닫힘 + 상단에 문장 표시

## 🎭 인터랙션 및 애니메이션

### 기본 원칙
- **부드럽게**: 모든 전환은 ease 곡선
- **빠르게**: 애니메이션 시간 0.2-0.3초
- **의미있게**: 필요한 곳에만 애니메이션

### 주요 애니메이션

#### 1. 페이지 로드
```
1. 헤더 페이드인 (0.3s)
2. 좌측 사이드바 슬라이드 인 (0.4s)
3. 우측 사이드바 슬라이드 인 (0.4s)
4. 중앙 안내 문구 페이드인 (0.5s)
```

#### 2. 카테고리 선택 애니메이션

**문장형 카테고리 (좌측) 선택 시**:
```
[Step 1] 클릭 (0~0.2s)
┌──────────────┐
│  물과 바람은  │  ← 문장 전체 하이라이트 (배경 gray-100)
│  하나가      │
│  되어서      │
└──────────────┘

[Step 2] 상단으로 이동 (0.2~0.6s)
헤더 아래 영역:
(아래에서 위로 슬라이드 인)
translateY(20px) → translateY(0)
opacity: 0 → 1

[Step 3] 점 표시 (0.4~0.6s)
     ˙
물과 바람은 하나가 되어서  ← 점(˙) 페이드인

[Step 4] 작품 카드 표시 (0.6~1.0s)
중앙 영역에 작품 카드들:
- 페이드인 + 위에서 아래로 (translateY)
- Stagger: 0.05s (순차적으로)
```

**텍스트형 카테고리 (우측) 선택 시**:
```
[Step 1] 클릭 (0~0.2s)
┌──────────────┐
│  디자인      │  ← 선택된 항목 하이라이트 (배경 gray-100)
│  조각        │
└──────────────┘

[Step 2] 작품 카드 표시 (0.2~0.6s)
중앙 영역에 작품 카드들:
- 페이드인 + 위에서 아래로
- Stagger: 0.05s
```

**CSS/Framer Motion 구현 예시**:
```typescript
// 문장형 카테고리 상단 표시 애니메이션

// 점(˙) 애니메이션
const dotVariants = {

const containerVariants = {

  }

const cardVariants = {
```
